# 신고 기능 테스트 가이드

신고 관련 4가지 기능의 테스트 방법을 안내합니다.

---

## 1. 중복 신고 방지

**테스트 방법**
1. 로그인 후 게시글 상세 페이지 진입
2. `🚨 신고` 버튼 클릭 → 사유 선택 → 신고하기
3. 같은 게시글에서 `🚨 신고` 버튼 다시 클릭 → 신고하기

**기대 결과**
- 첫 번째 신고: `신고가 접수되었습니다.` (초록 토스트)
- 두 번째 신고: `이미 신고한 내역이 있습니다.` (노란 토스트)

---

## 2. 신고 사유 표시 개선

**테스트 방법**
1. 로그인 후 게시글 상세 페이지 진입
2. `🚨 신고` 버튼 클릭
3. `기타` 선택 → 긴 텍스트 입력 → 신고하기
4. 관리자 계정으로 `/admin` 진입 → `🚨 신고 관리` 탭 확인

**기대 결과**
- 목록에서: 사유명(`기타 신고`)이 첫 줄, 상세 내용이 두 번째 줄에 말줄임 처리
- 상세 패널에서: 사유명과 전체 내용 분리해서 표시

---

## 3. 관리자 신고 상세 화면

**테스트 방법**
1. 관리자 계정으로 `/admin` 진입
2. `🚨 신고 관리` 탭 클릭
3. 신고 목록에서 항목 클릭

**기대 결과**
- 클릭한 항목 아래로 상세 패널이 펼쳐짐 (`▼ 상세` → `▲ 접기`)
- 상세 패널 내용: 신고 사유 / 신고 일시 / 처리 이력
- 처리 버튼: PENDING → `기각` `숨김` `삭제` / HIDDEN → `복구`
- 다른 항목 클릭 시 이전 항목 닫히고 새 항목 열림

---

## 4. 기타 사유 글자수 제한

**테스트 방법**
1. 로그인 후 `🚨 신고` 버튼 클릭
2. `기타` 선택
3. 입력창에 200자 이상 붙여넣기 시도

**기대 결과**
- 입력창 아래 `0 / 200` 글자수 카운터 표시
- 입력할수록 카운터 증가 (`50 / 200`, `100 / 200` ...)
- 200자 도달 시 카운터 빨간색으로 변경
- 200자 초과 입력 불가 (더 이상 입력되지 않음)
